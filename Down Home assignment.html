Down Home assignment

Time estimation: 3 hours
Migrating / deprecating an old data model in a sample Rails app.

Background

Picture is an old model we use to store all the User selected images in their profile. Here is a sample Picture :
Plain Text
Copy
user_id: 123, first_pic: 'http://foo.io/1.jpg', urls: 'http://foo.io/2.jpg,http://foo.io/3.jpg'
​
As you can see, we store all the image urls within a single row/record, the benefits are that we can have faster query time.
But as we want to do more advanced stuff like image resizing or image meta data, we realized that we need to migrate to a new model where each image is stored in its own row. For example, the above Picture would be stored as 3 rows / records:
Plain Text
Copy
user_id: 123, url: 'http://foo.io/1.jpg' user_id: 123, url: 'http://foo.io/2.jpg' user_id: 123, url: 'http://foo.io/3.jpg'
​

Requirements

creating a new ActiveRecord model PictureItem (with migration file) to store each picture separately
for example, currently, a user might have 3 pics urls stored in Picture
the corresponding PictureItem would be 3 of them, each storing a single url
implement 1 new endpoint /users_v2 to return the same response as /users but use only the newly created model (PictureItem)
also add a test case for this.
modifying the code (trigger from API 'update_user/:id') to support both Picture &PictureItem
whenever a user change Picture, it will also update/create PictureItem, so in the end, we have both tables "in sync" (containing the same set of data)
make sure all the rspec tests still passing by running bundle exec rspec
